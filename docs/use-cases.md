# EcoStay EMS — System Use Cases

This document outlines all the primary use cases in the EcoStay EMS platform, categorized by the "Actor" (who or what is performing the action). 

---

## 🧑‍💼 Admin Use Cases
*These actions are restricted to users with the `admin` custom claim and are performed in the `/admin` console.*

1. **Register a Room**: Add a new physical room to the system, give it a human-readable name, and assign it to a Property ID.
2. **Manage Owners**: Create a new Property Owner account, reset their access, or disable them.
3. **Assign Tenancy**: Assign an Owner account to a specific Property ID (which automatically grants them secure access to all rooms under that property).
4. **Configure Property Settings**: Set the CEB Tariff regime, define the rated wattages of controlled circuits (used for savings calculations), and set alert thresholds.
5. **Provision Devices**: Generate or reset secure, device-scoped passwords for the ESP32 hardware to authenticate with Firebase.
6. **Monitor Fleet Health**: See at a glance, across every property, which rooms' devices are reporting right now (last write within 15 s) and which safety alerts are currently open — then drill into the property detail to act. This is the operator half of the Admin role ("everything Owner can do, sees all properties" — CONTEXT.md); the per-property Alert Center and live room views are reached from the property detail.

---

## 🏨 Property Owner Use Cases
*These actions are performed by hotel owners monitoring their assigned properties via the main dashboard.*

1. **Secure Login**: Authenticate via email and password.
2. **Multi-Room Navigation**: View a list of all assigned rooms and seamlessly switch between them using the dashboard navigation.
3. **Monitor Live Telemetry**: View real-time data for a specific room (temperature, humidity, inferred occupancy state, power usage, gas levels, and water levels).
4. **Control Devices**: Remotely toggle room relays (lights, exhaust fans, water pumps, and motion detection tracking).
5. **Manage Routines**: Enable or disable automated routines for a room (e.g., turning on/off the Vacancy Cutoff automation).
6. **Analyze Energy & Cost**: View 24-hour and 7-day historical energy charts, the estimated month-to-date electricity bill, and financial savings generated by the system.
7. **Manage Alerts**: View, acknowledge, and dismiss active room safety alerts (e.g., device offline, high gas, high temperature).

## ⚙️ System (Automated) Use Cases
*These actions are executed by the serverless Vercel Cron jobs running in the background.*

1. **1-Minute Tick (Alerts & Automation)**:
   - Compare device heartbeat timestamps to server time to detect offline rooms and generate alerts.
   - Evaluate live telemetry against safety thresholds to generate Gas, Temperature, or Water alerts.
   - Execute the **Vacancy Cutoff automation**: automatically write "OFF" commands to lights and fans when a room transitions to `VACANT_CONFIRMED`.
2. **5-Minute Sampler**: 
   - Record a snapshot of the room's current power and energy usage for historical charting.
3. **Nightly Rollup (Billing & Pruning)**:
   - Aggregate the day's 5-minute samples into a single Daily Record (split by Time-of-Use buckets: Peak, Day, Off-Peak).
   - Calculate the daily avoided kWh (savings) based on how long the room was vacant and the rated wattage of the cutoff circuits.
   - Prune (delete) raw 5-minute samples older than 30 days to minimize database storage costs.

---

## 🔌 Firmware (ESP32 Device) Use Cases
*These actions are performed by the physical hardware in the hotel rooms.*

1. **Push Telemetry**: Write sensor data to the `latest` database path every 3 seconds.
2. **Poll Commands**: Read the `devices/*` command path every 500ms and actuate the physical relays accordingly.
